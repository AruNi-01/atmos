# Security Policy

Thank you for helping keep ATMOS and its users safe.

## Supported Versions

At this stage, we provide security fixes for the `main` branch and the most
recent tagged release line.

## Reporting a Vulnerability

Please **do not** open a public issue for sensitive vulnerabilities.

Use one of the channels below:

1. Preferred: email `hello@0x3f4.run`
2. Alternative: GitHub Security Advisories / private report, if enabled

When reporting, include:

- A clear description of the issue and impact
- Reproduction steps or proof of concept
- Affected components / paths
- Any suggested remediation

## Response Targets

- Initial acknowledgement: within 72 hours
- Triage decision: within 7 days
- Fix timeline: depends on severity and complexity; we will communicate updates

## Disclosure

We follow coordinated disclosure and will credit reporters unless anonymity is
requested.
