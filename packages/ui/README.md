## 📝 Component Categories

### 1. Atomic Components (`src/components/ui/`)

**DO NOT modify these directly**. These are auto-generated by shadcn CLI.

| Component | Purpose | Source |
|-----------|---------|--------|
| `button.tsx` | Button component | shadcn/ui |
| `card.tsx` | Card container | shadcn/ui |
| `input.tsx` | Form input | shadcn/ui |
| `dialog.tsx` | Modal dialog | shadcn/ui |
| `select.tsx` | Dropdown select | shadcn/ui |
| ... | ... | ... |

**Adding New**:
```bash
cd packages/ui
bun ui:add <component-name>
```

---

### 2. Coss UI Components (`src/components/coss/`)

Coss UI style components for specific design patterns.

**Adding New**:
```bash
cd packages/ui
bun ui:add:coss <component-name>
```

---

### 3. Animation Components (`src/components/animate/`)

Components with built-in animations.

**Adding New**:
```bash
cd packages/ui
bun ui:add:animate <component-name>
```

---

### 4. Custom Components (`src/components/`)

Business-agnostic shared components that can be used across apps.

Examples:
- `theme-toggle.tsx`: Dark/light mode toggle
- `language-selector.tsx`: Language switcher

**When to Add**:
- Component is used in 2+ apps
- Component is generic (not business-specific)
- Component doesn't fit in ui/coss/animate categories

## 🎨 Styles (`src/styles/globals.css`)

### Structure

```css
/* 1. Import Tailwind CSS v4 */
@import "tailwindcss";

/* 2. Define theme tokens */
@theme {
  --color-primary: oklch(...);
  --radius-lg: 0.5rem;
}

/* 3. CSS Variables for light/dark modes */
:root {
  --background: oklch(...);
  --foreground: oklch(...);
}

.dark {
  --background: oklch(...);
  --foreground: oklch(...);
}

/* 4. Base styles */
@layer base {
  * { @apply border-border; }
  body { @apply bg-background text-foreground; }
}
```

### Key Features

- **No `tailwind.config.ts`**: Uses Tailwind v4 CSS-based config
- **oklch Colors**: Modern color space for better gradients
- **CSS Variables**: For theme switching
- **Design Tokens**: Consistent spacing, colors, radius

## 🛠 Utilities (`src/lib/utils.ts`)

### `cn()` Function

Merge Tailwind classes with proper precedence:

```tsx
import { cn } from "@workspace/ui/lib/utils";

<Button className={cn("bg-blue-500", isActive && "bg-green-500")} />
```

### Other Utilities

Add shared utilities here:
- String formatting
- Number formatting
- Date utilities (if not in @vibe-habitat/shared)

## 🪝 Hooks (`src/hooks/`)

Shared React hooks used across apps.

**Examples**:
- `use-mounted.ts`: Check if component is mounted
- `use-media-query.ts`: Responsive breakpoints

**Usage**:
```tsx
import { useMounted } from "@workspace/ui/hooks/use-mounted";
```

## ⚙️ Configuration Files

### `components.json`

shadcn CLI configuration:

```json
{
  "style": "new-york",          // Component style
  "rsc": true,                  // React Server Components
  "tsx": true,                  // Use TypeScript
  "tailwind": {
    "config": "",               // No tailwind.config.ts (v4)
    "css": "src/styles/globals.css",
    "baseColor": "zinc",
    "cssVariables": true
  },
  "aliases": {
    "utils": "@workspace/ui/lib/utils",
    "ui": "@workspace/ui/components/ui",
    "lib": "@workspace/ui/lib",
    "hooks": "@workspace/ui/hooks",
    "components": "@workspace/ui/components"
  }
}
```

### `postcss.config.mjs`

PostCSS configuration for Tailwind v4:

```js
export default {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};
```

### `package.json`

Package exports:

```json
{
  "name": "@workspace/ui",
  "exports": {
    "./globals.css": "./src/styles/globals.css",
    "./postcss.config": "./postcss.config.mjs",
    "./lib/*": "./src/lib/*.ts",
    "./components/*": "./src/components/*.tsx",
    "./hooks/*": "./src/hooks/*.ts"
  }
}
```

## 🔄 Component Lifecycle

### 1. Installation (shadcn)

```bash
bun ui:add button
```

### 2. Auto-generated File

```
src/components/ui/button.tsx  # Created automatically
```

### 3. Usage in Apps

```tsx
import { Button } from "@workspace/ui/components/ui/button";
```

### 4. Customization (if needed)

Create wrapper in app:

```tsx
// apps/web/src/components/primary-button.tsx
import { Button } from "@workspace/ui/components/ui/button";

export function PrimaryButton({ ...props }) {
  return <Button className="bg-primary" {...props} />;
}
```
