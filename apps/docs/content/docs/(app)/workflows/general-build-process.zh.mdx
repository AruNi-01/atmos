---
title: 通用构建流程
description: 从创建工作空间到完成 Review 变更的日常流程。
---

在 Atmos 里，构建通常指的是在一个工作空间中，把想法变成可运行、已 Review 的变更。

默认流程很简单：创建工作空间，让 Agent 或人类完成修改，Review diff，运行应用，测试它，发布它，然后在任务结束后清理工作空间。

## 概览

大多数任务遵循这个顺序：创建工作空间、准备项目、用 Agent 或手动构建、Review 变更、运行、测试、发布，然后归档工作空间。

## 1. 打开或创建工作空间

参考[项目与工作空间管理](/zh/workflows/project-and-workspace-manager)：

- **Quick New Workspace** 适合快速开始。
- **New Workspace Composer**（`⌘N`）适合带 prompt、GitHub 链接或图片的任务。
- **Kanban** 适合整理大量任务。

开始编辑或运行 Agent 前，等待 **Workspace Setup Progress** 完成。

## 2. 运行 setup script

如果项目定义了 **Setup Script**，它会在创建工作空间时自动运行。常见用途包括：

- 安装项目依赖。
- 复制示例设置或同步密钥。
- 生成本地配置。

UI 中会显示 **Create Workspace**、**Run Setup Script** 和 **Ready** 等步骤。阻塞步骤完成后就可以继续。

在项目 **⋯** → **Workspace Scripts** 中配置脚本。

## 3. 在终端中用 Code Agent 构建

1. 打开工作空间的 **Terminal** 标签。
2. 在这个 shell 中启动你的编码 Agent。
3. 也可以使用**全局 Agent 聊天面板**处理不需要完整终端流程的聚焦任务。

给 Agent 一个清晰的目标、文件和约束。让它在工具中迭代，同时你在同一个面板里查看输出。

## 4. Review 代码（可选）

在信任 diff 之前：

1. 打开右侧边栏的 **Review** 标签。
2. 创建一个 **Review Session**。
3. 对 diff 留评论，或使用 Review Skill 运行 **Agent Review**。
4. 用 **Fix** 把未解决评论发回给 Code Agent。

完整流程见[在 Atmos 中做代码 Review](/zh/workflows/code-review-in-atmos)。

## 5. 手动调整（可选）

小修复可以使用**内置编辑器**，大型重构则使用你的主 IDE。编辑后在终端中重新运行测试。

## 6. 运行和预览

1. 如果还没有配置，先在项目上配置 **Run Script**。
2. 在工作空间中打开右侧边栏的 **Run / Preview** 标签。
3. 点击 **Run**（或 `⌘R`），在终端中执行 run script。
4. 当应用输出预览链接时，Atmos 可以在内置浏览器预览中打开它。

如果没有 run script，你会看到 **No Run Script Configured**，并可以跳转到 **Workspace Scripts**。

## 7. 测试、发布和清理

1. 在终端中运行项目测试，例如单元测试、端到端测试、lint，或你的仓库使用的检查。
2. 用正常 Git 流程提交或打开 PR；Atmos Git 视图可以帮助你先检查 diff。
3. 任务完成后，**Archive** 工作空间，可以从侧边栏或 Kanban 执行。
4. 如果配置了 **Purge Script**，它会删除较重的生成文件，避免旧工作空间占满磁盘。

**Settings** → **Workspace** 中的 **Archive Behavior** 控制确认方式，以及归档时如何处理终端会话。
