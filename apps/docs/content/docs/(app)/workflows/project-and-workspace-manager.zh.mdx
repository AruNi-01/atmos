**项目**是磁盘上的 Git 仓库；**工作区**是隔离的任务环境（通常为 **Git worktree**），拥有自己的分支、终端、编辑器、Agent 与审查历史。

## 概述

### 什么是工作区？

工作区是单一任务单元：功能、修复、实验或审查。终端、Code Agent、diff、Kanban 卡片都附着在工作区上。

### 为什么用 Git worktree 隔离？

Worktree 让多个工作区在同一仓库的**不同分支**上并行，互不干扰：

- 多 Agent 并行跑在不同分支。
- 任务之间上下文清晰分离。
- 切换任务无需重新 clone。

## 核心能力

| 能力 | 说明 |
|------|------|
| **Import Project** | 将 Git 仓库导入为项目 |
| **创建工作区** | 新 worktree + 创建/setup 流水线 |
| **Quick New Workspace** | 一键创建（自动命名/分支） |
| **Kanban** | 状态、优先级、标签 |
| **Workspace scripts** | Setup / Run / Purge 自动化 |
| **GitIgnore Dirs Sync** | 将忽略目录（如 `agents/`）同步进 worktree |

## 1. 添加项目

1. 左侧边栏底部 **Add Project**（文件夹图标），或全局搜索 / New Workspace 中的 **Add Project**。
2. 在 **Import Project** 中设置 **Project Path**（输入或 **Browse…** → **Select Project Directory**）。
3. 确认 Git 检测，设置 **Project Name**，点击 **Import Project**。

## 2. 配置 Workspace Scripts

项目 **⋯** → **Workspace Scripts**：

| 脚本 | 时机 |
|------|------|
| **Setup Script** | 新工作区创建后 |
| **Run Script** | 一键启动服务（Run / Preview，`⌘R`） |
| **Purge Script** | 删除/归档时清理 |

对话框中会说明脚本可用的环境信息（项目路径、工作区名称、工作区目录）。

## 3. GitIgnore Directories Sync

1. **Settings** → **Workspace** → **GitIgnore Directories Sync**。
2. 打开总开关。
3. 为路径选择 **Symlink** / **Copy** / **Off**。
4. 可添加自定义目录。

同步失败会提示 **GitIgnore Sync Failed**。

## 4. 创建工作区

### 4.1 侧边栏 Quick New Workspace

点击项目行 **Quick New Workspace**（⚡）→ 自动命名/分支 → 跑 setup → 进入工作区。

### 4.2 New Workspace Composer

1. **New Workspace**（管理中心 / `⌘N` / 项目菜单）。
2. 覆盖中间与右侧面板的 overlay。
3. **PromptComposer**：`@issue#`、`@pr#`、`@file:`、`/skill:`、粘贴图片；设置优先级/状态/标签。
4. **Advanced**（`⋯`）：名称、分支、GitHub Issue/PR。
5. **Create Workspace**（`⌘⇧↵`）→ setup → 打开工作区并可排队初始 Agent 提示。

### 4.3 Create New Workspace 对话框

从 Kanban 列 **+** 或 **Issue Only** 卡片 **Build workspace from issue** 打开；可关联 GitHub、**Auto-extract TODOs with LLM**。

### 4.4 Kanban：仅 Issue 工作区

1. **Import GitHub Issues** → 选择仓库与 issue → **Import**。
2. 卡片为 **GitHub Issue Only Workspace**（尚无 worktree）。
3. 就绪后 **Build workspace from issue** 创建真实工作区。

## 5. 用 Kanban 管理

列：**Backlog**、**To Do**、**In Progress**、**In Review**、**Blocked**、**Completed**、**Canceled**。

支持筛选、按项目/状态/时间分组、排序、**Show/Hide Issue Only**；拖拽改状态，**Pin** / **Archive** / **Delete**。
