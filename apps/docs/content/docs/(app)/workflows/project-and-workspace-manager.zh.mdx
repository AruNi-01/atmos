项目和工作空间是 Atmos 组织真实工作的方式。**项目**是磁盘上的代码库。**工作空间**是一个围绕单个任务的聚焦区域，有自己的分支、终端、编辑器、Agent 和 Review 历史。

## 概览

### 什么是工作空间？

工作空间是一项工作的最小单位：功能、修复、实验或 Review。你在 Atmos 中做的一切，包括终端、Code Agent、diff 和 Kanban 卡片，都挂在某个工作空间下面。

### 为什么要分离工作空间？

分离的工作空间让多个任务可以共享同一个项目，同时停留在不同分支上。这样可以：

- 在不同分支上并行运行 Code Agent。
- 清晰分离不同任务。
- 不重新克隆也能快速切换上下文。

## 核心能力

| 能力 | 作用 |
|------|------|
| **Import Project** | 把项目文件夹添加到 Atmos |
| **Create workspace** | 为任务创建一个聚焦工作空间 |
| **Quick New Workspace** | 一键创建带自动名称和分支的工作空间 |
| **Kanban** | 跨工作空间管理状态、优先级和标签 |
| **Workspace scripts** | 为每个项目配置 setup、run 和 purge 自动化 |
| **Ignored folder sync** | 把选定的忽略文件夹带入新工作空间 |

## 1. 添加项目

1. 在左侧边栏底部点击 **Add Project**（文件夹图标），或从全局搜索 / New Workspace composer 中使用 **Add Project**。
2. 在 **Import Project** 中选择 **Project Path**，可以手动输入，也可以点击 **Browse…** → **Select Project Directory**。
3. 确认 Git 检测结果，设置 **Project Name**，然后点击 **Import Project**。

项目会出现在侧边栏里，工作空间会显示在项目下方。

## 2. 配置工作空间脚本

每个项目都可以配置在关键生命周期中运行的脚本。打开项目 **⋯** 菜单 → **Workspace Scripts**。

| 脚本 | 运行时机 |
|------|----------|
| **Setup Script** | 新工作空间创建后运行，例如依赖、环境文件等 |
| **Run Script** | 一键启动应用或本地预览 |
| **Purge Script** | 删除或归档工作空间时清理文件 |

对话框会列出脚本可使用的环境值，例如项目路径、工作空间名称和工作空间文件夹。

完成后点击 **Save Scripts**。

## 3. 忽略文件夹同步

有些团队会把 Agent 指令、prompt 或本地工具放在不会提交到 Git 的文件夹里。如果新工作空间也需要这些文件夹，请启用同步。

1. 打开 **Settings** → **Workspace** → **GitIgnore Directories Sync**。
2. 打开主开关。
3. 对每个文件夹选择让 Atmos 链接、复制或忽略。
4. 需要时添加自定义文件夹。

如果同步失败，创建工作空间时 Atmos 会显示 **GitIgnore Sync Failed**。

## 4. 创建工作空间

### 4.1 Quick New Workspace（侧边栏）

1. 在项目行点击 **Quick New Workspace**（⚡）。
2. Atmos 会创建一个带自动名称和分支的工作空间，将状态设为 **In Progress**，并准备它。
3. 准备完成后，你会进入这个工作空间。

适合：没有 GitHub 链接或自定义分支名的快速探索。

### 4.2 New Workspace Composer

1. 从 Management Center（`⌘N`）、项目菜单或全局搜索打开 **New Workspace**。
2. **New Workspace** 覆盖中间和右侧面板，侧边栏仍然可见。
3. 在 prompt composer 中：
   - 输入目标，添加有用引用，或粘贴图片。
   - 设置 **Priority**、**Status**、**Labels**，并选择项目。
4. **Advanced**（`⋯`）：设置 **Name**、**Base branch**、**Current workspace branch**，或关联 **GitHub Issue** / **GitHub PR**。
5. 点击 **Create Workspace**；Atmos 会创建工作空间、准备它、打开它，并且可以排队第一个 Agent prompt。

适合：带有丰富上下文的 Agent-first 任务。

### 4.3 Create New Workspace 对话框（Kanban 和 issue）

1. 在 **Kanban** 中点击某一列上的 **+**，或在 **Issue Only** 卡片上点击 **Build workspace from issue**。
2. 在 **Create New Workspace** 或 **Build Workspace from Issue** 中选择项目、名称、分支，以及可选的 GitHub issue 或 pull request。
3. **Create Workspace** 会运行和 composer 相同的准备流程。

### 4.4 Kanban：Issue Only 工作空间

1. 打开 **Kanban**（侧边栏图标，或 `⌘⇧K` 展开看板）。
2. 点击 **Import GitHub Issues**，选择仓库，加载 issue，然后点击 **Import N Issue(s)**。
3. Issue 会以 **GitHub Issue Only Workspace** 卡片出现，在真正创建工作空间前先被看板跟踪。
4. 准备开始时，使用 **Build workspace from issue** 创建真实工作空间并开始构建。

## 5. 用 Kanban 管理工作空间

**列**表示工作流状态：**Backlog**、**To Do**、**In Progress**、**In Review**、**Blocked**、**Completed**、**Canceled**。

**工具栏：**

- 按状态、优先级、标签、项目过滤。
- 按 **By Project**、**By Status** 或 **By Time** 分组。
- 按 **Last Visit**、**Create Time** 或 **Priority** 排序。
- 切换 **Show Issue Only** / **Hide Issue Only**。

**卡片操作：**拖动改变状态，或执行 **Pin**、**Archive**、**Delete**，也可以打开工作空间。

用 Kanban 回答一个问题：*我接下来该做什么？*
