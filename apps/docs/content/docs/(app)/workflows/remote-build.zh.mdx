---
title: 远程构建
description: 在另一台机器上构建，同时从笔记本或浏览器使用 Atmos。
---

**远程构建** 指的是 Atmos UI 运行在一个设备上，而项目 runtime 运行在已注册的 **Atmos Computer** 上。

当另一台机器应该负责仓库、终端、Agent、工作空间状态和构建输出时，使用这个流程。

## 概览

连接到 Atmos Computer 后：

- 工作空间、分支、终端和 Agent 历史都属于那台电脑；
- 你的笔记本或浏览器只是客户端；
- 当 UI 已连接到那台电脑时，本地 CLI 命令也会解析到同一个活跃电脑。

要回到眼前这台机器的 runtime，选择 **Use locally**。

## 第 1 步：创建 Access Key

1. 打开 **Settings** → **Atmos Computer**。
2. 找到 **Access Key** 区域。
3. 生成新 key，或粘贴你已经信任的 key。
4. 将它保存在负责管理电脑的设备上。
5. 如果需要注册另一台 host，立即复制它。

请像保护密码一样保护 access key。

## 第 2 步：注册电脑

### 当前机器

1. 确保 Atmos 正在这台电脑上运行。
2. 打开 **Settings** → **Atmos Computer** → **This Computer**。
3. 启用注册。
4. 等待这台 host 出现在 **My Computers** 中。

### 另一台机器

1. 在负责运行工作的机器上安装 Atmos。
2. 在保存了 access key 的设备上创建注册码。
3. 在目标机器上运行界面显示的 CLI 命令。
4. 确认该电脑显示为 **Online**。

注册码是短期且一次性的。如果过期，请生成新的注册码。

## 第 3 步：连接

1. 打开 **Settings** → **Atmos Computer** → **My Computers**。
2. 选择一台在线电脑。
3. 点击 **Connect**。
4. 等待 UI 切换到那台电脑。

离线电脑会保留在列表中，但需要重新上线后才能选择。

## 第 4 步：正常工作

连接后，继续使用同样的 Atmos 工作流：

- [General Build Process](/zh/workflows/general-build-process)

## 安全注意事项

| 主题 | 说明 |
|------|------|
| **Access key** | 拥有 key 的人可以管理你注册的电脑，请妥善保存。 |
| **注册码** | 短期、一次性。删除未使用的注册码。 |
| **Session** | Connect session 会过期，需要时从 **My Computers** 重新连接。 |
| **撤销** | 当一台电脑不应该继续可用时，从 **My Computers** 中移除它。 |
