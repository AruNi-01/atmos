Choose the setup that matches where you want your work to run.

## Desktop app

The desktop app is the best starting point when your projects are on your own computer.

- On macOS, install with Homebrew using `brew install --cask AruNi-01/tap/atmos`.
- You can also use the macOS installer from `https://install.atmos.land/install-desktop.sh`.
- For Windows and Linux, download the latest build from [GitHub Releases](https://github.com/AruNi-01/atmos/releases).

After installing, open Atmos like any other app.

## Browser app

Use the browser app when you want the UI in a tab, or when you plan to connect to an Atmos Computer.

For local browser use, the local web runtime starts Atmos on your machine and opens a local address in your browser. Install it from the package `@atmos/local-web-runtime` or use the installer linked from the release notes.

## First workspace

1. Launch Desktop or open Atmos in your browser.
2. Add a project folder when Atmos asks for one.
3. Create a workspace for the task you want to run in parallel.
4. Atmos prepares an isolated working folder and branch for that task.
5. Open the workspace terminal.
6. Start your code-agent CLI in that terminal.
7. Review the diff before you commit.

## First Code Agent Terminal

Atmos is terminal-first: start the code agent you already use inside the workspace terminal. Each workspace keeps its own terminal session, branch, files, and diff, so you can run several agents side by side without mixing their work.

For parallel work, create one workspace per task and start one agent in each workspace terminal. Use the sidebar, Kanban, Canvas, and Review views to monitor progress and bring finished changes back into your normal Git flow.
