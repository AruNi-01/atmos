# Introduction

Atmos is a concurrent development workspace for running multiple code agents in persistent terminal sessions.

It gives each task its own Git worktree, keeps long-running code-agent terminals alive, and lets you switch between parallel branches without losing context. You can run Atmos as a desktop app, open it in a browser, or connect the app to another machine where your code actually lives.

## What You Can Do

- **Open several tasks at once** — create a separate workspace and branch for each feature, bug fix, experiment, or review.
- **Run multiple code agents in parallel** — keep one or more agent terminals alive per workspace.
- **Keep long-running work recoverable** — terminal sessions stay attached to the workspace across reloads, app restarts, and even machine reboots.
- **Work with agents where the context is** — run your code-agent CLI in the workspace terminal, then inspect files, diffs, reviews, and output in the same place.
- **Review before you ship** — inspect diffs, leave line comments, ask an agent to fix them, then commit with confidence.
- **Plan visually** — pin terminals and notes to Canvas when several workspaces need to be compared.
- **Track many tasks** — use Kanban, labels, priority, search, and agent status to stay oriented.
- **Work from another machine** — connect to an Atmos Computer when your repos and agents should run on a remote box.
