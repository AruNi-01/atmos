Review 工作流刻意放慢节奏——逐条阅读改动、留下行内评论，在提交或 Pull Request 前请 Agent 按评论修复。

## 为什么重要

- 在合并前发现 Agent 失误。
- 在精确行号记录审查意见。
- 无需把评论复制到其他工具。
- 高风险改动保留人工判断。

## 审查流程

1. 打开有待提交改动的工作区。
2. 打开 Git 或审查视图阅读 diff。
3. 在需要修改的行留下评论。
4. 请 Code Agent 根据评论修复。
5. 重新运行检查并复审直至通过。

## 关注什么

- 改动是否完成原始目标？
- 是否缺少测试、文档或边界情况？
- diff 能否更小、更清晰？

## 提示

- Agent 修复后运行项目测试。
- 与 [集成 Git 工作流](/zh/features/integrated-git-workflow) 配合提交与 PR。
