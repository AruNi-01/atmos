Review 工作流会有意让你慢下来：逐项阅读变更、留下行内评论，并在提交或打开 Pull Request 前让 Agent 处理这些问题。

## 为什么重要

- 在合并前发现 Agent 犯的错误。
- 把 Review 备注记录在具体代码行上。
- 不用把评论复制到其他地方，也能完成闭环。
- 在有风险的改动中保留人工判断。

## Review 流程

1. 打开包含待处理改动的工作空间。
2. 打开 Git 或 Review 视图，阅读 diff。
3. 在需要修改的行上留下评论。
4. 让 Code Agent 根据这些评论修复问题。
5. 重新运行检查并再次 Review，直到干净为止。

## 要看什么

- 这个改动是否解决了原始任务？
- 是否缺少测试、文档或边界情况？
- diff 是否还能更小、更清晰？

## 使用建议

- Agent 修复后运行项目测试。
- 和 [集成 Git 工作流](/zh/features/integrated-git-workflow) 搭配，完成提交和 PR 步骤。
