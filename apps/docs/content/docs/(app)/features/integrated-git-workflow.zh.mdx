---
title: 集成 Git 工作流
description: 在一个地方查看 diff、辅助提交、代码 Review 和管理 GitHub PR。
---

集成 Git 工作流把日常版本控制放在 Atmos 里：查看改了什么、准备提交、协调 Review，不用在多个工具之间来回切换。

## 为什么重要

- 在一个地方检查 Agent 或手动修改。
- 提交或打开 PR 前清楚地查看 diff。
- 减少终端、编辑器和 Git UI 之间的上下文切换。
- 在你对某些行留下评论后，支持 Review 和交给 Agent 修复。

## 你可以做什么

- 查看已修改文件和并排 diff。
- 根据工作空间上下文暂存和提交。
- 查看与当前任务相关的历史。
- 连接仓库后管理 GitHub Pull Request。

## 开始使用

1. 打开包含改动的工作空间。
2. 打开 **Git** 视图。
3. 逐个文件阅读 diff。
4. 在终端中运行项目检查。
5. 用你正常的 Git 流程提交或打开 PR。

## 使用建议

- 每次 Agent 运行后都 Review diff，不要只在最后看。
- 使用 [Review Workflow](/zh/features/review-workflow) 做行内评论和 Agent 修复。
- 保持提交较小，并和工作空间任务名对应。
