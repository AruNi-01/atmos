title: 集成 Git 工作流
description: Diff 视图、提交辅助、代码审查与 GitHub PR 管理一体化。
---

集成 Git 工作流把日常版本管理留在 Atmos 内——查看改动、准备提交、协调审查，减少在终端、编辑器与 Git 工具间来回切换。

## 为什么重要

- 一处查看 Agent 或人工改动。
- 提交或开 PR 前看清 diff。
- 降低上下文切换成本。
- 支持行内评论后交给 Agent 修复。

## 能做什么

- 查看变更文件与并排 diff。
- 结合工作区上下文暂存与提交。
- 查看与当前任务相关的历史。
- 连接仓库后管理 GitHub Pull Request。

## 快速开始

1. 打开有改动的工作区。
2. 打开 **Git** 视图。
3. 逐文件阅读 diff。
4. 在终端运行项目检查。
5. 按惯用流程提交或开 PR。

## 提示

- 每次 Agent 运行后都看一眼 diff，不要只在最后审查。
- 配合 [Review 工作流](/zh/features/review-workflow) 做行内评论与修复。
- 提交粒度与工作区任务名保持一致。
