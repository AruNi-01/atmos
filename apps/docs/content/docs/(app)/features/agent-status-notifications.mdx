---
title: Agent Status Notifications
description: Get alerts when agents finish, fail, or need your attention.
---

Agent status notifications tell you when something needs attention, so you do not have to watch a terminal the whole time.

Use notifications for long-running tasks, approval prompts, and failures. Keep them selective: the goal is to know when to come back, not to turn every small update into an alert.

## Why it matters

- Walk away while long agent jobs run.
- Get pinged when approval is required.
- Know immediately when a task fails.
- Route alerts to phone or team channels via self-hosted push.

## Supported channels

- Desktop or system notifications.
- Push services such as **ntfy**, **Gotify**, or a custom webhook when configured.

## Get started

1. Open **Settings** in Atmos.
2. Find the notifications section.
3. Enable the channels you will actually act on.
4. Run a short agent task to confirm delivery.

## Tips

- Enable only alerts you respond to—noise hides important events.
- Pair with [Agent Status Tracking](/features/agent-status-tracking) for in-app state sync.
