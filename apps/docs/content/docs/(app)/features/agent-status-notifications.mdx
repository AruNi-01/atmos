---
title: Agent Status Notifications
description: Hook-based alerts when agents finish, fail, or need permission—native and self-hosted push.
---

Agent status notifications tell you when something needs attention without watching the terminal. Atmos listens to agent hooks and can deliver alerts through system notifications or your own push stack.

## Why it matters

- Walk away while long agent jobs run.
- Get pinged when approval is required.
- Know immediately when a task fails.
- Route alerts to phone or team channels via self-hosted push.

## Supported channels

- Native desktop or OS notifications.
- Self-hosted push via **ntfy**, **Gotify**, or a custom webhook.

## Get started

1. Open **Settings** in Atmos.
2. Find the notifications section.
3. Enable the channels you will actually act on.
4. Run a short agent task to confirm delivery.

## Tips

- Enable only alerts you respond to—noise hides important events.
- Pair with [Agent Status Tracking](/features/agent-status-tracking) for in-app state sync.
