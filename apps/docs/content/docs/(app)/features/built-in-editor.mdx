title: Built-in Lightweight Editor
description: Preview files, make inline edits, and switch back to manual coding anytime.
---

The built-in editor is for quick inspection and small edits inside Atmos—without leaving your workspace, terminal, or agent flow. Open a file, adjust a few lines, and return to the terminal or chat when you are done.

## Why it matters

- Read code next to terminal output and agent replies.
- Fix typos or small logic changes without opening another IDE.
- Stay in one surface for task-focused work.
- Switch to your main editor anytime for large refactors.

## What you can do

- Browse the workspace file tree.
- Preview syntax-highlighted source files.
- Edit inline for focused changes.
- Jump from diffs or agent suggestions into the file.

## Get started

1. Open a workspace.
2. Select a file from the project tree or from a Git diff.
3. Read or edit in the built-in editor.
4. Run checks in the terminal before you commit.

## Tips

- Use Atmos for review-sized edits; use your primary IDE for multi-file rewrites.
- Pair with [Integrated Git Workflow](/features/integrated-git-workflow) to inspect changes before commit.
