---
title: Lightweight Local Models
description: Use small local models for lightweight helper tasks.
---

Lightweight local models let Atmos handle small helper tasks on your own machine instead of sending every request to a cloud model.

## Why it matters

- Extract task lists from workspace notes locally.
- Draft Git commit messages from the diff on device.
- Keep light inference off subscription quotas.

## Good fits

- Short text generation and classification.
- Summaries of small inputs.
- Repetitive helpers that do not need frontier models.

## Not a replacement for

- Large refactors or multi-file reasoning.
- Tasks that need the latest frontier models—use your Code Agent CLI for those.

## Get started

1. Open model or local inference settings in Atmos.
2. Choose a small model and start it when available.
3. Use local generation for titles, task extraction, or commit message drafts.
4. Fall back to cloud agents for heavy coding work.

## Tips

- Prefer quantized small models for speed on laptops.
- Verify GPU/CPU requirements before enabling on battery power.
