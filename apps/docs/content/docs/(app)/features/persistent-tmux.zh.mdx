---
title: Tmux 持久化会话
description: 基于 tmux 的容错终端，会话在中断与应用重启后仍可恢复。
---

持久化 tmux 会话让终端工作始终附着在工作区上——即使界面重载、网络抖动或暂时离开，长时间运行的服务、安装与 Agent 命令仍可继续。

## 为什么重要

- 切换视图时 dev server 与测试不会意外退出。
- Agent 驱动的命令保持在稳定的 shell 上下文中。
- 关闭再打开 Atmos 后快速回到原会话。
- 在同一工作区内运行多个 shell 而不丢上下文。

## 工作原理

Atmos 通过 **tmux** 管理终端。每个工作区终端对应一个 tmux 会话，面板与滚动历史在中断后仍然保留。

## 快速开始

1. 打开工作区。
2. 打开 **终端** 面板。
3. 运行常用的 dev、测试或 Agent 命令。
4. 切换到编辑器、Canvas 或其他工作区——回来时仍是同一会话。

## 提示

- 多个终端时建议命名会话。
- 若输出卡住，在该 pane 停止进程后于同一工作区重跑。
- 与 [多工作区开发](/zh/features/multi-workspace) 配合，在并行分支上跑 Agent。
