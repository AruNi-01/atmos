---
title: 持久终端会话
description: 让终端工作始终挂在对应工作空间上。
---

持久终端会话会把命令行工作挂在对应工作空间上。即使界面刷新、应用重启、连接断开，甚至电脑宕机重启，你也可以从同一个工作空间恢复终端，而不是从空白终端重新开始。

## 为什么重要

- 应用预览、测试运行和 Agent 会话在中断后可以恢复。
- Agent 执行的命令停留在稳定的 shell 上下文中。
- 关闭 Atmos 或重启电脑后也能快速恢复。
- 在一个工作空间中运行多个 shell，也不会丢失位置。

## 它如何工作

当你重新打开工作空间时，Atmos 会重新连接到已保存的终端会话，而不是从空白终端开始。

## 开始使用

1. 打开一个工作空间。
2. 打开 **Terminal** 面板。
3. 运行你常用的开发、测试或 Agent 命令。
4. 切到编辑器、Canvas 或另一个工作空间；回来时仍然是同一个会话。

## 使用建议

- 如果同时打开多个终端，给会话起名字。
- 如果输出看起来卡住了，在那个面板里停止进程，然后从同一个工作空间重新运行。
- 当 Agent 在多个分支上并行工作时，和 [多工作空间开发](/zh/features/multi-workspace) 搭配使用。
