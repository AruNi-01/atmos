---
title: 多工作空间开发
description: 并排运行多个编码任务，而不混在一起。
---

多工作空间开发让你同时保留多个编码任务，而不会混淆终端、Agent、文件或 Review 备注。每个工作空间都是围绕一个目标的聚焦区域。

## 为什么重要

- 在不同分支上并行运行 Agent。
- 让实验远离主项目文件夹。
- 切换任务时不需要关闭终端，也不会丢失会话状态。
- 让一个工作空间对应一个目标，例如 `fix-login` 或 `docs-refresh`。

## 它如何工作

当项目使用 Git 时，Atmos 会为每个工作空间创建一个独立工作文件夹。这样多个工作空间可以共享同一个仓库，同时停留在不同分支上。

每个工作空间都有自己的终端、编辑器标签、diff、Agent 历史和 Review 状态。

## 开始使用

1. 打开 Atmos，选择 **New Workspace**。
2. 选择一个项目文件夹或仓库。
3. 为这个任务选择或创建分支。
4. 用正在做的事情给工作空间命名。
5. 打开终端或 Agent 面板开始工作。

## 使用建议

- 一个任务对应一个工作空间；发布后归档或关闭。
- 使用简短、可搜索的名称，让 Kanban 和搜索更好用。
- 大型工作可以把规划、实现和 Review 拆到不同工作空间中。
