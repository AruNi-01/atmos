---
title: 多工作区开发
description: 基于 Git worktree 隔离，在多个分支或实验上并行运行任务。
---

多工作区开发让你同时推进多个编码任务，而不会混用终端、Agent 或文件上下文。每个工作区对应一个聚焦环境，绑定到项目目录或 Git worktree。

## 为什么重要

- 在不同分支上并行运行 Agent。
- 将实验与主工作树隔离。
- 切换任务时不必关掉终端或丢失会话状态。
- 一个工作区对应一个目标，例如 `fix-login` 或 `docs-refresh`。

## 工作原理

Atmos 使用 **Git worktree 隔离**，让同一仓库的不同分支拥有独立工作区。每个工作区保留自己的终端、编辑器视图、Git diff 与 Agent 历史。

## 快速开始

1. 打开 Atmos，选择 **新建工作区**。
2. 选择项目文件夹或仓库。
3. 选择或创建该任务对应的分支或 worktree。
4. 用任务名命名工作区。
5. 打开终端或 Agent 面板开始工作。

## 提示

- 尽量一个工作区一个任务，完成后归档或关闭。
- 使用简短、可搜索的名称，便于 Kanban 与全局搜索。
- 大型任务可拆成规划、实现、审查多个工作区。
