---
title: Atmos Computer
description: 将 Atmos 连接到另一台运行项目的机器。
---

**Atmos Computer** 让你的桌面应用或浏览器连接到另一台机器，项目、终端、Agent 和工作空间都在那台机器上运行。

当你想在更强的机器、家里的机器或云主机上构建，同时把 Atmos 界面留在笔记本上时，可以使用它。

这和 [跨平台与远程访问](/zh/features/cross-platform-remote-access) 里的 **Remote Access** 是分开的；Remote Access 使用外部隧道分享**你当前所在的这台机器**。

## Atmos Computer 与 Remote Access

| | **Atmos Computer** | **Remote Access**（隧道） |
|---|-------------------|---------------------------|
| **工作运行在哪里** | 你注册的那台电脑 | 这台电脑 |
| **连接方式** | Atmos 将应用连接到那台电脑 | 隧道把这台电脑发布出去 |
| **客户端需要隧道工具吗** | 不需要 | 需要，在负责托管的桌面端上 |
| **主要入口** | **Atmos Computer** → **Connect** | **Remote Access** → **Public URL** |
| **项目存放在哪里** | 远程机器 | 这台机器 |

有些界面在 **This Computer** 上会出现 “remote” 字样。在这个区域里，它的意思是“让这台电脑可以作为 Atmos Computer 使用”，不是“启动 Remote Access 隧道”。

## 为什么重要

- 把仓库、密钥和构建放在你控制的基础设施上。
- 从笔记本、台式机或浏览器使用一台更强的远程机器。
- 避免在笔记本上额外配置隧道工具。

## 连接如何工作

远程机器会和 Atmos 保持连接。当你点击 **Connect** 时，你的应用会通过同一个 Atmos 服务加入那台机器。日常使用 Atmos 时，你不需要知道那台机器的公网地址。

## 典型流程

1. **Access Key** — 在 **Settings → Atmos Computer → Access Key** 创建并保存。
2. **Register** — 把远程机器添加到你的账户，可以在那台机器上操作，也可以使用注册码。
3. **Connect** — 在 **My Computers** 下选择电脑；Atmos 会把界面切换到那台机器。
4. **Work** — 工作空间、终端、Canvas 和 Agent 都在**那台**电脑上运行。
