# Project Wiki Workflow

## Stages

1. `Index`: build repo and concept indexes.
2. `Research`: understand subsystem boundaries and design intent.
3. `Plan`: create page registry and page plans.
4. `Assemble Evidence`: create one evidence bundle per page.
5. `Write`: generate final Markdown pages.
6. `Audit`: validate page quality against plans and evidence.

## Principles

- Prefer evidence over style quotas.
- Prefer page-specific bundles over repo-wide prompt dumps.
- Prefer adaptive navigation over a hard-coded section tree.
- Keep compatibility exports derived, never primary.
